# Contribuindo

## Fluxo recomendado

1. Crie uma branch a partir de `main`.
2. Use nomes como `feature/revista-digital` ou `fix/pdf-a5`.
3. Faça commits pequenos e descritivos.
4. Execute `npm run build` antes de abrir o Pull Request.
5. Abra um Pull Request explicando alterações e testes realizados.

## Padrões

- TypeScript para toda a aplicação.
- Componentes reutilizáveis em `components/`.
- Conteúdo editorial centralizado em `content/`.
- Não armazenar chaves do Supabase no repositório.
